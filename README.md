# 🎬 CineMatch — AI Movie Recommendation System

A production-grade, content-based movie recommender built with **Python**, **Scikit-learn**, and **Streamlit** — featuring TMDB posters, ratings, and a cinematic dark UI.

---

## ✨ Features

| Feature | Details |
|---|---|
| **Content-Based Filtering** | Cosine Similarity on combined movie features |
| **5,000+ Movies** | TMDB 5000 Movie Dataset |
| **TMDB Integration** | Live posters, ratings, overviews & genres |
| **Search Dropdown** | Autocomplete across all 5,000 titles |
| **Cinematic Dark UI** | Playfair Display typography, gold/crimson accent |
| **Error Handling** | Graceful fallbacks for missing movies/posters |
| **Deploy-Ready** | Works on Streamlit Community Cloud out of the box |

## 🚀 Quick Start

### Step 1 — Clone / Download this project

```bash
git clone https://github.com/your-username/cinematch.git
cd cinematch
```

### Step 2 — Create a virtual environment

```bash
python -m venv venv
# macOS / Linux
source venv/bin/activate
# Windows
venv\Scripts\activate
```

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 4 — Download the dataset

1. Go to **[Kaggle — TMDB 5000 Movie Dataset](https://www.kaggle.com/datasets/tmdb/tmdb-movie-metadata)**
2. Download and extract the ZIP
3. Place both files inside a `data/` folder:
   ```
   data/
   ├── tmdb_5000_movies.csv
   └── tmdb_5000_credits.csv
   ```

### Step 5 — Add your TMDB API key (optional but recommended)

1. Sign up at [themoviedb.org](https://www.themoviedb.org)
2. Go to **Settings → API** and copy your API key
3. Create `.env` from the template:
   ```bash
   cp .env.example .env
   # Then edit .env and paste your key
   ```

> **Without the key**: The app still works — recommendations are shown without posters/ratings.

### Step 6 — Train the model

```bash
python model.py
```

This will:
- Load and merge the two CSVs
- Parse genres, keywords, top 5 cast members, and director
- Build a 5,000-word vocabulary with CountVectorizer
- Compute the 5,000×5,000 cosine-similarity matrix
- Save `artifacts/movies.pkl` and `artifacts/similarity.pkl`

Expected output:
```
📂 Loading datasets…
   → 4,803 movies loaded after merge
🔧 Preprocessing…
   → 4,800 movies after cleaning
🔢 Vectorising tags…
   → Vector matrix: (4800, 5000)
📐 Computing cosine similarity…
   → Similarity matrix: (4800, 4800)
✅ Artifacts saved to 'artifacts/'

🧪 Quick test — recommendations for 'Avatar':
   1. Guardians of the Galaxy
   2. Aliens
   3. Star Wars
   4. Titan A.E.
   5. Space Jam
```

### Step 7 — Launch the app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## 🧠 How It Works — Step by Step

### 1. Data Loading
Two TMDB CSVs are merged on the `id` column, keeping: `title`, `overview`, `genres`, `keywords`, `cast`, `crew`.

### 2. Feature Engineering
Each movie is converted into a **bag of tokens** by combining:

| Source | Example tokens |
|---|---|
| `overview` | `"stranded", "planet", "war"` |
| `genres` | `"Action"`, `"ScienceFiction"` |
| `keywords` | `"spacetravel"`, `"aliens"` |
| `cast` (top 5) | `"SamWorthington"`, `"ZoesSaldana"` |
| `crew` (director) | `"JamesCameron"` |

Spaces are removed from multi-word names so they're treated as single tokens.

### 3. Vectorisation
`CountVectorizer` builds a **5,000-word vocabulary** and converts each movie's tag string into a vector of word counts.

### 4. Cosine Similarity
`cosine_similarity()` computes pairwise similarity for all 4,800 movies, producing a 4,800×4,800 matrix.

### 5. Recommendation
Given a query movie, its row in the similarity matrix is retrieved, sorted in descending order, and the top 5 results (excluding itself) are returned.

## ☁️ Deploy to Streamlit Community Cloud

1. Push your code to GitHub (**don't commit `data/`, `artifacts/`, or `.env`**)
2. In GitHub Actions or locally: run `model.py` and commit the `artifacts/` folder separately, OR use Git LFS
3. Go to [share.streamlit.io](https://share.streamlit.io) → **New app** → select your repo
4. Under **Advanced settings → Secrets**, add:
   ```toml
   TMDB_API_KEY = "your_key_here"
   ```
5. Click **Deploy**

> **Note:** `artifacts/` can be large (~100 MB). For cloud deploy, consider committing them to the repo using Git LFS, or generating them in a setup script.
